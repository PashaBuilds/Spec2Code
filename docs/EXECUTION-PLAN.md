# Spec2Code ‚Äî Execution Plan & Decision Log

This repository implements the **Spec2Code Build Brief** verbatim. The brief is the
authoritative spec. This file records (a) the build phasing and (b) every place where
the brief left a decision open and we picked a default ‚Äî per Brief ¬ß23.7
("pick a reasonable default, flag it in a code comment, and proceed").

## Build phasing (follows Brief ¬ß4)

- **A ‚Äî Foundation (sequential):** skeleton + git + contracts (`project.spec`,
  `descriptor`, `ruleset` schemas) + `hostplat/` isolation module.
  *Verify:* CRLF write + tool-resolve self-test pass.
- **B ‚Äî Deterministic core:** `xparameters.h` parser ‚Üí 4 sample headers + 4 platform
  topologies + 4 seed descriptors (LTC2991 / MT25Q128 / TCA9548A / MT25QU02G) ‚Üí Jinja2
  static codegen + catalog. *Verify:* real `ltc2991.c/.h` + test, CRLF, naming, mux
  channel-select injected, MT25QU02G `0xB7` 4-byte init differs from MT25Q128.
- **C ‚Äî QC loop:** runners (clang-format / clang-tidy / cppcheck) + naming-linter
  (libclang AST, reads ruleset) + loop (N=3, graceful-degrade, WS broadcast, report).
- **D ‚Äî Backend (FastAPI):** REST + WS, generate job.
- **E ‚Äî Frontend:** Vite + Tailwind + shadcn + React Flow schematic, catalog,
  protocol-aware device params, generate console, CodeMirror code view, driver import.
- **F ‚Äî Pluggable LLM layer:** client + tasks + LLM-in-QC (toggle, default OFF).
  *Verify:* 2026-06-26 local Ollama OpenAI-compatible smoke (`gemma3:12b`) fixed a QC
  gate error and re-ran clean; `optimize_code()` returned non-empty C output.
- **G ‚Äî README:** macOS dev setup + Windows smoke-test checklist.

## Decisions / defaults (flagged)

1. **`platform/` ‚Üí `hostplat/`.** The brief names the isolation module `platform/`.
   A top-level Python package named `platform` **shadows the stdlib `platform` module**
   (uvicorn / httpx / asyncio all `import platform`), a guaranteed import-collision bug ‚Äî
   exactly the "port surprise" the brief warns against (¬ß1.1). Renamed to `hostplat/`.
   The isolation property (one and only place for platform-specific code) is unchanged.
2. **CRLF discipline.** All generated `.c/.h/.md` output is written **only** through
   `hostplat.io.write_output()`, which normalizes every line ending to `\r\n` and writes
   in binary mode. No other code writes generated output.
3. **Tool resolution & graceful degradation.** `hostplat.tools.resolve(name)` searches
   PATH + known install dirs (Homebrew llvm, CommandLineTools, common Windows dirs) +
   env override `SPEC2CODE_<TOOL>_PATH`. If a QC tool is missing the QC loop emits a
   structured warning and **skips that check** instead of hard-failing ‚Äî required for the
   air-gapped target and for partial dev setups. The naming-linter (libclang) always runs.
4. **Codegen maps named operations, not raw registers.** A mux-attached device gets a
   channel-select call injected before every device access (Brief ¬ß10, ¬ß13).
5. **RAG (¬ß17) deferred** to a later pass (its own "last" priority + Python 3.14 wheel risk
   for torch/faiss). `rag/` ships as a clean stub; nothing in the deterministic path
   depends on it. Fallback = hand-authored descriptors + direct `.c/.h` exemplars.
6. **LLM config** comes from env / config: `SPEC2CODE_LLM_BASE_URL`, `SPEC2CODE_LLM_MODEL`,
   `SPEC2CODE_LLM_API_KEY`, `SPEC2CODE_LLM_TIMEOUT_S`, `SPEC2CODE_LLM_MAX_TOKENS`,
   `SPEC2CODE_LLM_MAX_RESPONSE_CHARS`, `SPEC2CODE_LLM_RETRIES`. The model name is supplied
   exactly by the user; Kimi, Qwen, or any other OpenAI-compatible model uses the same field.
   Default `enabled: false`.
7. **Sample `xparameters.h`** for all four platforms are generated by us (no real file on
   hand yet); the same regex parser consumes a real file unchanged when provided.
8. **PoC exclusions honored (¬ß21):** no real compile, no git integration, single core,
   no full Vitis scaffold. Code left open for a future "compile" QC runner.

## How to run (filled in as phases land)

- Backend: `uvicorn backend.main:app --reload` (from repo root).
- Frontend: `cd frontend && npm install && npm run dev`.
- Deterministic codegen self-test: `python -m orchestrator.selftest` (Phase B).
